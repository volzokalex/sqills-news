## How to refresh the dataset

```bash
npm run refresh
# or:  node refresh.mjs
```

What happens:

1. `refresh.mjs` calls each source adapter (Anthropic = HTML via cheerio; DeepMind +
   Hugging Face = RSS via `rss-parser`). Each adapter returns a normalized item shape.
2. Top **4 most recent items per source** are kept (configurable: `PER_SOURCE_LIMIT`).
3. For every kept item, the script fetches the article URL once more and extracts
   `og:image` (with fallbacks: `og:image:secure_url`, `twitter:image`, etc.). 3
   in-flight at a time (`OG_IMAGE_CONCURRENCY`).
4. All items sorted reverse-chrono by `date_iso`.
5. `news.js` is regenerated with a banner + `const NEWS_DATA = {...}`.
6. Refuses to overwrite if zero items came back (safety against blanking the proto).

Adding a source = add a row to the `SOURCES` array, point at the adapter type
(`html` or `rss`). HTML scrapers may need source-specific selectors â€” copy
`adapterAnthropic` and adapt.

No env vars, no auth, no secrets. All three sources are public.

## What this prototype is NOT

- Not connected to Strapi or any backend.
- **Manual-trigger refresh** â€” `node refresh.mjs` runs on demand; nothing schedules it
  yet (GitHub Actions cron is the next step).
- Not AI-rewriting content â€” bodies are placeholder text + attribution + outbound link
  to the original publisher.
- Not deployed to GitHub Pages (yet).
- Not pulling real Figma component renderings â€” DS values are **approximated** from
  the markdown; final port should resolve every visual value through Figma MCP per the
  DS v2 resolution protocol (`design-system-v2.md` "How AI uses this file").
- Hotlinks images from the publisher's CDN â€” fine for Anthropic / DeepMind / HF
  (PR-friendly publishers). Adding a media source like TechCrunch / Wired in the
  future will likely need self-hosting (download into `images/` during refresh) to
  dodge Referer-blocks and URL rot.

## Format-unit still open

Proto is built as **continuous story stream**. The other two formats on the table:

- **Hybrid** â€” same stream but each item tagged Brief (60-sec) or Deep (rich article);
  card visually differs by type.
- **Daily issue** â€” one dated digest per day with named sections; feed = list of issues.

Decide on a real screen before locking. The current proto is a good "stream" reference;
a "hybrid" variant could be tested by adding a `length: "brief" | "deep"` field to each
item and a second card layout.
